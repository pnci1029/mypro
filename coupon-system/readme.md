````
    1. RDB에 지속적인 부하 발생 (요청이 큰 경우)
    
    2. 동시에 많은 양을 처리하는 도중 다른 인서트 동작이 발생할 경우 타임아웃 문제 발생 가능
        -> ex) insrt를 1분에 10000건 가능한 상황에 100,000건의 인서트 요청이 들어왔을 경우,
               그 사이 회원가입이나 주문 insert가 요청되었을때 100,000건의 요청이 전체가 처리되지 않을 수 있다.
````
